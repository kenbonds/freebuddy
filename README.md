# FreeBuddy AI Community
### 全球多语言AI社区 · 智能论坛 & 好友私聊 & 全维度风控系统

## 📖 项目简介
FreeBuddy AI Community 是一款**面向全球用户的国际化智能社区系统**，基于 React + TypeScript + 本地AI模型开发。
内置全球无限语种双向自动翻译、分区论坛交流、好友私聊体系、全维度智能风控，适配跨国团队、多语言用户无障碍沟通。

## ✨ 核心功能
### 🌐 全球国际化多语言体系
- 系统UI原生支持：简体中文 / 英语 / 日语 / 韩语 / 印地语
- **用户发言支持全球无限语种**，任意语言自动翻译为浏览者母语
- 原文永久留存，千人千面独立展示，双向无障碍互通

### 📁 互联网分类论坛
- 办公交流专区：日常办公、协作、事务沟通
- 开发交流专区：技术开发、代码、项目问题交流
- 分区独立数据，场景精准隔离

### 👥 好友 & 私聊系统
- 支持添加好友、好友列表管理
- 一对一私密私聊对话
- 私聊内容同步风控校验，杜绝私下违规传播

### 🛡️ 企业级全维度风控系统（核心亮点）
- 全域关键词黑名单：广告、引流、灰产、色情、暴力、恐怖、诈骗违禁词全覆盖
- AI语义智能审核，识别谐音、变体、隐晦违规内容
- 用户行为限流、重复内容拦截、防机器人灌水
- 三级梯度处罚：提醒拦截 / 临时禁言 / 永久禁言+IP永久封禁
- IP段封禁、代理IP高危校验
- 全链路操作审计日志，可追溯、可合规复盘

### 🤖 AI模型预置配置
- 支持本地 Ollama 模型 / 远程 OpenAI 模型一键切换
- 标准化模型参数预置，开箱即用
- 离线翻译能力，无外网依赖、无额度限制

## 🧰 技术栈

## 🚀 快速安装部署教程（新手零基础可运行）
### 环境依赖
- Node.js 16+ 版本
- npm / yarn 包管理工具
- 可选：Ollama（本地AI翻译能力，无外网也可运行）

### 1. 克隆项目到本地
```bash
git clone https://github.com/kenbonds/freebuddy.git
cd freebuddy
```

### 2. 安装项目依赖
```bash
npm install
```

### 3. 启动本地开发项目
```bash
npm run start
```
启动成功后，打开浏览器访问：`http://localhost:3000`

### 4. AI翻译功能部署（核心能力）
#### 方式一：本地 Ollama 离线部署（推荐，免费无额度限制）
1. 下载安装 Ollama：[https://ollama.com/](https://ollama.com/)
2. 本地终端拉取模型：
```bash
ollama pull llama3
```
3. 项目默认适配本地接口，**无需修改配置**，直接拥有离线多语言翻译能力

#### 方式二：远程 OpenAI 部署
1. 进入模型配置文件，填写自己的 OpenAI Key
2. 切换模型类型为 `remote`，即可使用远程AI翻译服务

### 5. 项目打包上线
```bash
npm run build
```
打包后的 `build` 文件夹可直接部署至服务器、静态网站平台。

## ❗ 常见启动报错解决方案（全覆盖避坑）
### 报错1：node版本过低 / 语法报错
**问题原因**：Node.js 版本低于16，不支持TS新语法
**解决方案**：
- 升级 Node.js 至 `16.x / 18.x / 20.x` 稳定版
- 升级后重新执行 `npm install`

### 报错2：npm install 依赖安装失败、报错、卡死
**问题原因**：外网源超时、依赖缓存冲突
**解决方案**：
```bash
npm cache clean --force
npm config set registry https://registry.npmmirror.com
npm install
```

### 报错3：端口3000被占用、启动失败
**问题原因**：本地已有程序占用3000端口
**解决方案**：
```bash
# Windows
netstat -ano | findstr :3000
taskkill /F /PID 对应进程号

# Mac/Linux
lsof -i :3000
kill -9 对应进程号
```

### 报错4：AI翻译无响应、接口请求失败
**问题原因**：Ollama未启动、未拉取模型、本地接口异常
**解决方案**：
1. 终端执行 `ollama ps` 检查服务状态
2. 执行 `ollama pull llama3` 安装基础模型
3. 确认本地接口 `http://127.0.0.1:11434` 可正常访问
4. 关闭本地代理、翻墙工具，避免接口拦截

### 报错5：项目启动白屏、无报错
**解决方案**：
- 强制刷新页面 `Ctrl+F5`
- 清空浏览器缓存，重启本地项目

### 报错6：TS类型报错、模块找不到
**解决方案**：
1. 删除 `node_modules`、`package-lock.json`
2. 重新执行 `npm install`

### 报错7：打包build失败
**解决方案**：
```bash
npm run clean
npm run build
```
