## 🚀 快速安装部署教程（新手零基础可运行）
### 环境依赖
- Node.js 16+ 版本
- npm / yarn 包管理工具
- 可选：Ollama（本地AI离线能力）

### 1. 克隆项目到本地
```bash
git clone https://github.com/kenbonds/freebuddy.git
cd freebuddy
```

### 2. 安装项目依赖
```bash
npm install
```

### 3. 启动本地开发项目
```bash
npm run start
```
启动成功后，打开浏览器访问：`http://localhost:3000`

### 4. AI功能部署（完全自由选择）
#### 方式一：本地 Ollama 离线部署（推荐，永久免费无限制）
1. 下载安装 Ollama：[https://ollama.com/](https://ollama.com/)
2. 本地终端拉取模型：
```bash
ollama pull llama3
```
3. 开箱即用，无需复杂配置，**完全离线自由使用**

#### 方式二：远程 OpenAI 部署
1. 填写个人 OpenAI Key
2. 一键切换远程模型，自由切换部署方案

### 5. 项目打包上线
```bash
npm run build
```
打包产物可直接部署至任意服务器、静态平台，**完全自主掌控、无限制**。

## ❗ 常见启动报错解决方案（全覆盖避坑）
### 报错1：node版本过低 / 语法报错
**解决方案**：
- 升级 Node.js 至 `16.x / 18.x / 20.x` 稳定版
- 升级后重新执行 `npm install`

### 报错2：npm install 依赖安装失败、报错、卡死
**解决方案**：
```bash
npm cache clean --force
npm config set registry https://registry.npmmirror.com
npm install
```

### 报错3：端口3000被占用、启动失败
**解决方案**：
```bash
# Windows
netstat -ano | findstr :3000
taskkill /F /PID 对应进程号

# Mac/Linux
lsof -i :3000
kill -9 对应进程号
```

### 报错4：AI无响应、接口请求失败
**解决方案**：
1. 终端执行 `ollama ps` 检查本地服务
2. 执行 `ollama pull llama3` 安装模型
3. 检查本地接口地址，关闭代理拦截

### 报错5：项目白屏、缓存异常
**解决方案**：
- `Ctrl+F5` 强制刷新、清空浏览器缓存重启项目

### 报错6：模块找不到、TS报错
**解决方案**：
1. 删除 `node_modules`、`package-lock.json`
2. 重新安装依赖

### 报错7：打包失败
**解决方案**：
```bash
npm run clean
npm run build
```
